# SEO Indexing — Owner Setup Checklist

This site auto-generates `sitemap.xml` on every push and can ping IndexNow so
Bing/Yandex (and increasingly Google) re-crawl within minutes of a deploy. The
build pipeline is fully automated; the items below are **one-time owner setup**
done in the browser.

- **Site:** https://virarprint.in (GitHub Pages, custom domain via `CNAME`)
- **Sitemap:** https://virarprint.in/sitemap.xml (generated by `scripts/build-sitemap.js`)
- **Scripts:** `scripts/build-sitemap.js`, `scripts/ping-indexnow.js`

---

## 1) Google Search Console (one-time)

1. Go to **https://search.google.com/search-console** and sign in.
2. Add a property → choose the **URL prefix** type → enter `https://virarprint.in`.
3. Pick the **HTML tag** verification method. Google shows a tag like:

   ```html
   <meta name="google-site-verification" content="REPLACE_WITH_GSC_TOKEN" />
   ```

4. The orchestrator places that tag inside `index.html` **`<head>`** (high up,
   near the other `<meta>` tags). Replace `REPLACE_WITH_GSC_TOKEN` with the exact
   token Google gives you — keep the rest of the tag verbatim.
5. Push to `main`, wait for GitHub Pages to deploy, then click **Verify** in GSC.
6. In GSC → **Sitemaps**, submit: `https://virarprint.in/sitemap.xml`

> The verification meta tag stays in `index.html` permanently — do not remove it
> after verifying, or Google may drop the verification.

---

## 2) Bing Webmaster Tools (one-time)

1. Go to **https://www.bing.com/webmasters** and sign in.
2. **Easiest path:** click **Import** and import the site directly from Google
   Search Console — this carries over verification and the sitemap automatically.
3. If importing isn't available, add `https://virarprint.in` manually and verify
   with the **HTML meta tag** method (same `<head>` placement pattern as GSC):

   ```html
   <meta name="msvalidate.01" content="REPLACE_WITH_BING_TOKEN" />
   ```

4. Submit the sitemap: `https://virarprint.in/sitemap.xml`

---

## 3) IndexNow (no account needed)

IndexNow has **no dashboard and no sign-up**. Ownership is proven purely by the
key file hosted at the site root:

- **Key:** `f2a5c7686a8f412b0e9e78465d440d6e`
- **Key file:** `https://virarprint.in/f2a5c7686a8f412b0e9e78465d440d6e.txt`
  (contents are exactly the key; generated/kept in sync by `ping-indexnow.js`)

`scripts/ping-indexnow.js` reads `sitemap.xml` and submits all URLs to
`https://api.indexnow.org/indexnow`. It is **fail-safe** — any network error
logs a warning and exits `0`, so it can never break the build or a deploy.

Make sure the `*.txt` key file is committed and served by GitHub Pages, then run
the ping **after** the site has finished deploying (so the pinged URLs and the
key file are actually live).

---

## Quick reference

```bash
npm run build:sitemap   # regenerate sitemap.xml
npm run ping            # submit sitemap URLs to IndexNow (safe to re-run)
node scripts/ping-indexnow.js --dry-run   # build payload + key file, no network
```
